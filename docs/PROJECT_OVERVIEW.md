# Project Overview

このファイルは、プロジェクトの目的と全体像を説明するためのテンプレートです。

初めて読む人、人間、ChatGPT、Codexの誰が見ても、このプロジェクトが何を目指しているのか分かるように書いてください。

## プロジェクト名

例:

```text
UTENA Task Manager
```

## 一言でいうと

例:

```text
個人事業のタスク、請求、締切を一元管理するMac用ローカルアプリ。
```

## 目的

このプロジェクトで達成したいことを書きます。

記入例:

- 毎日の作業予定を迷わず確認できるようにする
- 請求漏れを防ぐ
- ChatGPTとCodexで継続開発できるように、仕様と現在地をドキュメント化する

## 作る理由

なぜこのソフトウェアが必要なのかを書きます。

記入例:

```text
現在はタスク管理、請求管理、メモが別々の場所にあり、見落としが起きやすい。
1つの画面で「今日やること」と「期限が近いこと」を確認できるようにしたい。
```

## 想定ユーザー

記入例:

- タカシ
- UTENAの業務を手伝うメンバー
- 将来このプロジェクトを引き継ぐ開発者

## 主要機能

記入例:

| 機能 | 説明 | 状態 |
| --- | --- | --- |
| タスク一覧 | 今日・今週のタスクを表示する | 未着手 |
| 締切通知 | 期限が近いタスクを強調する | 未着手 |
| データ保存 | ローカルにデータを保存する | 未着手 |

## 作らないこと

スコープ外のことを書きます。

記入例:

- 最初の版ではクラウド同期は作らない
- 最初の版ではスマホアプリは作らない
- 会計ソフトとの自動連携は後回しにする

## 成功条件

このプロジェクトが「使える」と判断できる条件を書きます。

記入例:

- アプリを開くと今日やることがすぐ分かる
- データが消えない
- 主要操作を説明なしで使える
- READMEとdocsを読めば、次に何をすればいいか分かる
