# Current Status

このファイルは、プロジェクトの現在地を説明するためのテンプレートです。

会話履歴を知らない人が読んでも、今どこまでできていて、何が未完成なのか分かるように書いてください。

## 最終更新日

```text
YYYY-MM-DD
```

## 現在の状態

記入例:

```text
初期設計中。まだアプリ本体は未実装。
READMEとdocsのテンプレートだけ作成済み。
```

## 完了していること

未確認のものは書かないでください。  
確認済みのものだけを書きます。

記入例:

- GitHubリポジトリを作成済み
- READMEとdocsを作成済み
- 基本画面のワイヤーフレームを決定済み

## 作業中のこと

記入例:

- タスク一覧画面の実装
- データ保存方式の検討

## 未完成のこと

記入例:

- タスク追加機能
- タスク編集機能
- データのバックアップ
- Macアプリ化

## 未確認のこと

未確認なのに「完了」と書かないための欄です。

記入例:

- 長時間起動したときにデータが壊れないか未確認
- Mac再起動後にデータが復元されるか未確認
- 1000件以上のタスクで重くならないか未確認

## 次にやること

必ず書いてください。

記入例:

1. タスク一覧画面を作る
2. タスク追加フォームを作る
3. ローカル保存を実装する
4. 実際にアプリを開いて動作確認する

## 最新の判断メモ

なぜその状態になっているのか、判断理由を書きます。

記入例:

```text
まずはクラウド同期なしのローカル版で作る。
理由は、最初に必要なのは自分の作業管理であり、共有機能は後から追加できるため。
```
